# Sasha and a Very Easy Test

# 解题思路

因为模数 PPP 不一定是质数，不一定存在逆元，所以除法的操作需要特殊处理。

考虑一个数只有与模数 PPP 不互质时才没有逆元，我们可以考虑将 PPP 质因数分解，那么一个数被分成两部分：与 PPP 互质的部分 和 与 PPP 不互质的部分。与 PPP 互质的部分是 PPP 中不包含的质因数 (的乘积)，不互质是包含的部分。

互质的部分除就直接乘逆元，不互质的部分需要记录一个数所含 PPP 的各个质因子的个数，一个数的值将这两部分乘起来就可以，除法的话直接指数相减再更新答案。

# Squirrel Migration

# 解题思路

考虑对于每一条边的最大贡献，对于树上的一条边，把它切断的话，树会分成两个联通块，分别记为 S1S_1S1​ 和 S2S_2S2​ ，设 ∣S1∣<∣S2∣|S_1| < |S_2|∣S1​∣<∣S2​∣。

现在有了结论，就可以容斥求答案了。

设 f(i)f(i)f(i) 表示至少有 iii 个点的 pxp_xpx​ 属于 iii 所在子树的答案，则题目所求为 ∑i=1n(−1)i∗f(i)∗(n−i)!\sum\limits _{i=1}^{n} (-1)^i*f(i)*(n-i)!i=1∑n​(−1)i∗f(i)∗(n−i)! ， 单独考虑每一个子树 subtree(y)subtree(y)subtree(y)，设其子树大小为 siz(y)siz(y)siz(y)，则在这个子树中 f(i)=(xi)2∗i!f(i) = \binom{x}{i}^2*i!f(i)=(ix​)2∗i!， 最终以 GGG 为根的 fff 就是把所有子树使用背包合并起来，时间复杂度 O(n2)O(n^2)O(n2)。

考虑到这个子树背包的形式与多项式乘法一样，我们可以将每个子树的 fff 看成一个多项式，每次选出两个长度最短的多项式相乘，因为所有多项式的总长度为 nnn，所以时间复杂度是 O(nlog⁡2n)O(n \log^2 n)O(nlog2n) 的。(有点类似于启发式合并)。

# Cigar Box

# 解题思路

考虑一个合法的操作序列，一个数可能被操作过很多次，但只有最后一次有用，我们定义一个数的最后一次操作为一个关键操作。设 f(i,l,r()f(i,l,r()f(i,l,r() 为进行了 iii 次操作，其中有 lll 次是放到左边的关键操作， rrr 次是放到右边的关键操作的方案数，于是我们就有了一个 DP:f(i,l,r)=f(i−1,l,r)∗(l+r)+f(i−1,l−1,r)+f(i−1,l,r−1)f(i,l,r) = f(i-1,l,r)*(l+r) + f(i-1,l-1,r) + f(i-1,l,r-1)f(i,l,r)=f(i−1,l,r)∗(l+r)+f(i−1,l−1,r)+f(i−1,l,r−1) 。但时间复杂度 O(mn2)O(mn^2)O(mn2) ， 原地爆炸。
